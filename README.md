### 常見類型（Type）

以下是本專案中使用的常見提交類型：

* **feat**: 新增功能。
* **fix**: 修復錯誤。
* **docs**: 文件變更（例如 `README.md`）。
* **style**: 不影響程式碼執行的格式變更（例如：空白、分號、程式碼風格）。
* **refactor**: 程式碼重構，既不修復 Bug，也不新增功能。
* **perf**: 效能最佳化。
* **test**: 新增或修改測試。
* **chore**: 其他雜項變更，例如建構流程、輔助工具的變更。
